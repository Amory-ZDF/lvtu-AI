## 2. 导航系统

### 2.1 三层导航

| 层级 | 组件 | 作用 |
|------|------|------|
| **全局导航** | sidebar | 首页 / 社区 两个入口 |
| **决策流导航** | back-link + steps-bar | 线性前进/后退（开始→目的地→对比） |
| **详情子导航** | sub-tabs | 行程详情内 4 面板：行程/穿搭/机位/打包 |

### 2.2 侧边栏

```
🏠 首页   → nav('home')
💬 社区   → nav('community')
```

- 桌面：左侧固定 200px
- ≤900px：隐藏，汉堡按钮 → 左滑抽屉 + 半透明遮罩
- Logo 点击 → 首页
- 遮罩点击 → 关闭抽屉

### 2.3 页面路由表

| page ID | 页面 | 入口 |
|---------|------|------|
| `page-home` | 首页 | 默认 / 侧边栏 / Logo |
| `page-start` | 开始行程 | 首页 CTA / 我的行程「规划新行程」 |
| `page-destinations` | 目的地推荐 | 开始行程 → AI 生成 |
| `page-comparison` | 方案对比 | 目的地页「查看方案对比」 |
| `page-trip-detail` | 行程详情（工作台） | 对比页确认 / 首页行程卡片 / 社区帖子 / `#trip-detail` |
| `page-community` | 社区 | 侧边栏 |

### 2.4 核心导航函数

```javascript
nav(page)                // 切换全局页面，更新侧边栏高亮
openTripDetail(tripId)   // 打开行程详情 + 重置子 Tab 为 itinerary
switchSubTab(sub)        // 切换行程详情子面板
updateSidebar(page)      // 侧边栏高亮；null=全部取消
```

## 7. 核心组件说明

### 7.1 偏好滑块 (.pref-sliders)

3 条 0-100 range input，拖动时右侧数字实时更新：
```
🌿 自然风光  [========●=====]  75
🍜 美食探索  [=====●========]  55
🏛️ 人文历史  [===●=========]  35
```
- 滑块完全自定义样式（appearance:none + 自定义 track/thumb）
- `min-width:0` 确保可拖到极值
- JS: `input` 事件 → `nextElementSibling.textContent = this.value`

### 7.2 自然语言修改框 (.nl-modify)

位于行程面板顶部：
```
┌──────────────────────────────────────────────┐
│ 💬  想调整行程？直接告诉我…              [→] │
└──────────────────────────────────────────────┘
```
- focus-within → brand 边框 + 微光晕
- 回车或点击 → 按钮 → Toast "AI 正在调整..."
- 响应式：≤540px 缩小字号和内边距

### 7.3 目的地长预览 (.dest-preview)

3 列并排，每个卡片从上到下：
- dp-hero（大图 + 匹配度角标）
- 标题 + 地点 + 价格
- 风格标签 + 推荐理由
- 行程预览（D1/D2 badge）
- 亮点机位（4 个名称）
- 双按钮（方案对比 / 直接生成）

### 7.4 精简添加按钮 (.add-btn)

```
+ 添加行程点    （"+" 由 CSS ::before 提供）
```
- 无边框、透明背景
- hover → brand 色 + brand-ghost 背景

### 7.5 详情弹窗 (.detail-panel)

**触发方式**：
- 点击机位卡片 → `openSpotDetail(id)`
- 点击穿搭卡片 → `openOutfitDetail(id)`
- 在行程路线中点击景点旁的 📸/👗 按钮 → 直接弹对应详情

**弹窗样式**：居中模态框（560px 宽），scale+translate 弹性入场动画，半透明遮罩点击关闭，body 滚动锁定。

**弹窗内容**（机位）：
- 大图 → 出片率% + 难度星级 → 📍位置 → 📐构图指南 → 👗穿搭建议 → 交叉链接「查看完整穿搭方案 →」
- 交叉链接：关闭弹窗 → 跳到穿搭 Tab → 自动打开对应穿搭弹窗

**弹窗内容**（穿搭）：
- 大图 → 场景+天气 → 👗单品清单 → 💡穿搭理由 → 交叉链接「查看匹配机位 →」

数据存储在 `spotData`（4 个）和 `outfitData`（6 个）JS 对象中，按 ID 索引。

### 7.6 精简添加按钮 (.add-stop-btn)

行程面板底部：虚线边框卡片式按钮 `⊕ 添加行程点`
