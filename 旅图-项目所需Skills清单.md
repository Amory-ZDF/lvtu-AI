# 旅图项目所需 Skills 清单

## 1. 文档目的

这份文档用于整理 `旅图` 项目当前最值得使用的 `Skill`，方便后续在产品规划、PRD 重构、页面设计、前端实现、后端推进和项目拆解时快速选择合适的能力。

说明：
- 这里列的是 **这个项目真正高频需要的 Skill**
- 不是把当前所有已安装 Skill 全部罗列出来
- 优先保留和 `旅图` 当前阶段最相关的能力

---

## 2. 当前项目最核心的 Skills

### 2.1 产品与需求梳理

#### `prd-development`
- 用途：输出完整 PRD
- 适用场景：
  - 重构产品需求文档
  - 补齐背景、目标用户、功能模块、验收标准
  - 把原始需求整理成研发可执行文档
- 对旅图的价值：
  - 适合持续维护 `旅图-WebH5-重构PRD`
  - 是当前项目最核心的产品文档 Skill 之一

#### `jobs-to-be-done`
- 用途：梳理用户真实任务和动机
- 适用场景：
  - 分析用户为什么要旅行规划
  - 理清“去哪儿、怎么玩、怎么出片、怎么准备”的真实需求
- 对旅图的价值：
  - 能帮助旅图和普通旅行规划产品拉开差异
  - 适合在做产品定位、用户价值阐述时使用

#### `problem-framing-canvas`
- 用途：先定义问题，再设计解决方案
- 适用场景：
  - 梳理“用户真正卡住的问题是什么”
  - 判断需求是功能问题、信息问题还是决策问题
- 对旅图的价值：
  - 很适合旅图这种“决策型产品”
  - 能帮助避免把产品做成单纯的路线生成工具

#### `opportunity-solution-tree`
- 用途：把目标、机会、方案、验证路径串起来
- 适用场景：
  - 明确产品目标
  - 拆解新增功能，比如 `穿搭推荐`、`机位推荐`
  - 做增长或体验优化的机会树
- 对旅图的价值：
  - 适合把复杂功能拆成清晰的产品路径

#### `customer-journey-map`
- 用途：梳理用户旅程
- 适用场景：
  - 画用户从“种草 -> 决策 -> 规划 -> 行前准备 -> 行中执行 -> 行后分享”的完整路径
- 对旅图的价值：
  - 和当前 PRD 中的用户旅程、用户路径设计高度匹配

---

### 2.2 战略与定位

#### `positioning-statement`
- 用途：输出清晰的产品定位
- 适用场景：
  - 对外介绍产品
  - 路演、答辩、汇报、商业计划书
- 对旅图的价值：
  - 非常适合总结“旅图和普通旅行规划产品的区别”

#### `proto-persona`
- 用途：输出目标用户画像
- 适用场景：
  - 梳理年轻女性、都市白领、特种兵旅行用户画像
  - 统一设计和产品对目标用户的理解
- 对旅图的价值：
  - 有助于明确推荐策略和内容设计方向

#### `product-strategy-session`
- 用途：从定位、用户、问题到方案的整合式策略梳理
- 适用场景：
  - 做版本路线、阶段目标、差异化战略
- 对旅图的价值：
  - 当项目从“原型阶段”进入“正式产品推进”时会非常有用

#### `roadmap-planning`
- 用途：做路线图和阶段规划
- 适用场景：
  - 规划 `MVP / Beta / 正式版`
  - 区分当前先做什么，后续接 AI、Agent、数据库做什么
- 对旅图的价值：
  - 适合和当前后端阶段性开发配合使用

#### `prioritization-advisor`
- 用途：做需求优先级排序
- 适用场景：
  - 判断 `目的地推荐`、`路线生成`、`社区`、`穿搭`、`机位` 谁先做
- 对旅图的价值：
  - 能帮助控制项目范围，避免一次做太多

---

### 2.3 需求拆解与交付

#### `user-story`
- 用途：把功能拆成用户故事和验收标准
- 适用场景：
  - 把 PRD 进一步拆给前后端开发
- 对旅图的价值：
  - 很适合后续拆解：
    - 目的地推荐
    - 行程工作台
    - 打包清单
    - 社区

#### `user-story-mapping`
- 用途：围绕用户流程拆用户故事地图
- 适用场景：
  - 梳理主路径和版本切片
- 对旅图的价值：
  - 适合把当前 PRD 中的路径变成研发任务地图

#### `user-story-splitting`
- 用途：把大功能拆成小任务
- 适用场景：
  - 拆大模块，例如：
    - 行程生成
    - 穿搭推荐
    - 机位推荐
    - 社区种草同款规划
- 对旅图的价值：
  - 对研发排期很有帮助

#### `workshop-facilitation`
- 用途：组织结构化讨论
- 适用场景：
  - 开产品讨论会
  - 开需求评审会
  - 开设计 workshop
- 对旅图的价值：
  - 当你和设计、前端、后端一起推进时很好用

---

### 2.4 设计与前端表达

#### `taste-skill`
- 用途：生成更有设计感的前端原型
- 适用场景：
  - 输出旅图的高保真 UI
  - 做 landing page、产品展示页、页面重构
- 对旅图的价值：
  - 适合做更强视觉风格的方案版本

#### `impeccable`
- 用途：优化界面设计、信息层级、交互细节、页面质感
- 适用场景：
  - UI 重构
  - 页面润色
  - 交互细节优化
- 对旅图的价值：
  - 非常适合做旅图这种强调“出片感”和内容表达的产品

  - `customer-journey-map`
